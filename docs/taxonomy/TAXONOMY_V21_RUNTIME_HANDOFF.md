# Taxonomy v2.1 Runtime Handoff

## Authoritative source
- Proposal spec (governing source): `docs/taxonomy/trustcare_taxonomy_v2_1_proposal.json`
- Build generator: `scripts/taxonomy_v21_build.py`

The proposal file is the only source for canonical IDs, English display labels, aliases, and concern mappings.

## Generated runtime resources
Generator output (shipping runtime path):
- Base entities: `TrustCare/Resources/TaxonomyV21/base/taxonomy_v21_base_en.json`
- Concern domains: `TrustCare/Resources/TaxonomyV21/concerns/taxonomy_v21_concern_domains_en.json`
- Locale labels: `TrustCare/Resources/TaxonomyV21/labels/taxonomy_v21_locale_labels_<locale>.json`
- Locale aliases: `TrustCare/Resources/TaxonomyV21/aliases/taxonomy_v21_aliases_<locale>.json`

Compatibility-only fallback files:
- `TrustCare/Resources/taxonomy_v21_canonical_en.json`
- `TrustCare/Resources/taxonomy_v21_labels_en.json`
- `TrustCare/Resources/taxonomy_v21_symptom_concern_en.json`

## Active runtime behavior (source of truth)
Primary runtime read path:
- `TaxonomyCatalogStore` loads `TaxonomyV21/base`, `TaxonomyV21/concerns`, `TaxonomyV21/labels`, `TaxonomyV21/aliases`.
- `TaxonomyService` local-searches and local-browses from `TaxonomyCatalogStore` first.
- `TaxonomyService.searchTaxonomy` filters RPC results to canonical IDs when local corpus exists.

This ensures proposal-driven v2.1 files are the active source of truth for browse/search/display when local corpus is present.

## Fallback behavior (explicit)
Allowed fallback is intentionally narrow:
- Base fallback: `taxonomy_v21_canonical_en.json` (legacy English compatibility only)
- Concern fallback: `taxonomy_v21_symptom_concern_en.json` (legacy English compatibility only)
- Labels fallback: `taxonomy_v21_labels_en.json` only when locale is `en`
- Aliases fallback: `taxonomy_v21_aliases_en.json` only when locale is `en`
- `TaxonomyI18nLoader` reads only from `TaxonomyI18n/<locale>.json` and no longer scans root locale files.

Non-English locale labels/aliases do not fallback to legacy root files.

## Locale import workflow (no translation in this step)
Importer script:
- `scripts/taxonomy_v21_import_locale.py`

Safety properties:
- Canonical-ID keyed only
- Locale `en` import is blocked
- Missing IDs fail
- Unknown IDs fail
- Duplicate JSON keys fail
- Empty labels fail
- Deterministic key ordering in written output

Supported payload formats:
1) `entries` format (recommended)
```json
{
  "entries": [
    {
      "canonical_id": "SPEC_CARDIOLOGY",
      "label": "...",
      "aliases": ["..."]
    }
  ]
}
```
2) `labels`/`aliases` object format
```json
{
  "labels": {
    "SPEC_CARDIOLOGY": "..."
  },
  "aliases": {
    "SPEC_CARDIOLOGY": ["..."]
  }
}
```

Run import:
```sh
make taxonomy-v21-import-locale-dry-run LOCALE=tr INPUT=path/to/translated_tr.json
make taxonomy-v21-import-locale LOCALE=tr INPUT=path/to/translated_tr.json
```

## Validation and build before shipping
Run:
```sh
python3 scripts/taxonomy_v21_build.py
swift tools/taxonomy_v21_validate.swift
xcodebuild -scheme TrustCare -destination 'platform=iOS Simulator,name=iPhone 15' build
supabase db push --dry-run
```
